1. 사용자 관리 API

1.1. 사용자 목록 조회

모든 사용자의 목록을 페이지네이션, 검색, 필터링 기능과 함께 조회합니다.

요청

GET /api/v1/admin/members?page=0&size=10&searchKeyword=test&status=ACTIVE HTTP/1.1
Host: i13d202.p.ssafy.io
Query Parameters
ParameterTypeDescription

page

Number

페이지 번호 (0부터 시작, optional)

size

Number

페이지 당 항목 수 (optional)

searchKeyword

String

닉네임 또는 이메일 검색어 (optional)

status

String

ACTIVE 또는 INACTIVE (optional)

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "content": [
    {
      "memberId": 1,
      "nickname": "testNickname1",
      "email": "test_get_1753367017086@example.com",
      "birthDate": "1999-01-07T00:00:00",
      "gender": "MALE",
      "status": "ACTIVE"
    }
  ],
  "totalPages": 1,
  "totalElements": 1,
  "size": 10,
  "number": 0
}

1.2. 사용자 정보 수정 (관리자)

관리자가 특정 사용자의 정보를 수정합니다. 비밀번호는 수정 대상이 아닙니다.

요청

PUT /api/v1/admin/members/{memberId} HTTP/1.1
Host: i13d202.p.ssafy.io
Content-Type: application/json

{
  "nickname" : "updatedNickname",
  "birthDate" : [1999, 1, 8, 0, 0],
  "gender" : "FEMALE",
  "status" : "INACTIVE"
}

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "memberId": 1,
  "nickname": "updatedNickname",
  "email": "test_get_1753367017086@example.com",
  "birthDate": "1999-01-08T00:00:00",
  "gender": "FEMALE",
  "status": "INACTIVE"
}

1.3. 사용자 상태 변경

사용자 계정을 비활성화(논리적 삭제) 처리합니다. (기존 API 문서와 동일)

요청

PATCH /api/v1/members/{memberId}/status HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "memberId" : 2,
  "email" : "test_verified_1754024182049@example.com",
  "nickname" : "testNickname",
  "birthDate" : "1999-01-07T00:00:00",
  "gender" : "MALE",
  "status" : "INACTIVE"
}

2. 글귀 관리 API

2.1. 글귀 목록 조회 (관리자)

모든 글귀 목록을 페이지네이션, 검색, 정렬 기능과 함께 조회합니다.

요청

GET /api/v1/admin/quotes?page=0&size=10&searchKeyword=짜장면&sort=views,desc HTTP/1.1
Host: i13d202.p.ssafy.io
Query Parameters
ParameterTypeDescription

page, size

Number

페이지네이션 (optional)

searchKeyword

String

책 제목 또는 저자 검색어 (optional)

sort

String

views,desc 또는 createdDate,desc (optional)

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "content": [
    {
      "quoteId": 19,
      "quoteImageName": "image_name.jpg",
      "bookTitle": "짜장면 불어요!",
      "author": "이현(지은이), 윤정주(그림)",
      "quoteViews": 181
    }
  ],
  "totalPages": 1,
  "totalElements": 1
}

2.2. 글귀 삭제 (관리자)

관리자가 특정 글귀를 영구적으로 삭제합니다.

요청

DELETE /api/v1/admin/quotes/{quoteId} HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 204 No Content

3. 큐레이션 관리 API

3.1. 큐레이션 목록 조회

관리자 페이지에서 저장된 모든 큐레이션의 목록과 노출 순서를 조회합니다.

요청

GET /api/v1/admin/curations HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 200 OK
Content-Type: application/json

[
  {
    "id": "c1a2b3c4-d5e6-f7g8-h9i0-j1k2l3m4n5o6",
    "order": 1,
    "title": "한강 작가의 도서를 만나봐요",
    "description": "2024년 노벨문학상 수상 작가, 한강의 대표적인 작품들을 모아봤어요.",
    "quoteCount": 3,
    "bookCount": 2
  }
]

3.2. 큐레이션 순서 변경

드래그 앤 드롭으로 변경된 큐레이션 목록의 순서를 서버에 저장합니다.

요청

PATCH /api/v1/admin/curations/order HTTP/1.1
Host: i13d202.p.ssafy.io
Content-Type: application/json

{
  "orderedIds": [
    "p7q8r9s0-t1u2-v3w4-x5y6-z7a8b9c0d1e2",
    "c1a2b3c4-d5e6-f7g8-h9i0-j1k2l3m4n5o6"
  ]
}

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "message": "큐레이션 순서가 성공적으로 업데이트되었습니다."
}

3.3. 큐레이션 상세 조회

특정 큐레이션을 편집하기 위해 모든 상세 정보를 조회합니다.

요청

GET /api/v1/admin/curations/{curationId} HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "c1a2b3c4-d5e6-f7g8-h9i0-j1k2l3m4n5o6",
  "title": "한강 작가의 도서를 만나봐요",
  "description": "2024년 노벨문학상 수상 작가...",
  "items": [
    {
      "id": 101, "type": "book", "title": "채식주의자", "author": "한강",
      "coverUrl": "http://.../cover.jpg"
    },
    {
      "id": 205, "type": "quote", "content": "이제 그럴 수 없다는 절망에...",
      "author": "한강", "sourceBookTitle": "희랍어 시간"
    }
  ]
}

3.4. 큐레이션 생성

새로운 큐레이션을 생성하여 서버에 저장합니다.

요청

POST /api/v1/admin/curations HTTP/1.1
Host: i13d202.p.ssafy.io
Content-Type: application/json

{
  "title": "새로운 큐레이션",
  "description": "이것은 테스트 큐레이션입니다.",
  "items": [
    { "id": 101, "type": "book" },
    { "id": 205, "type": "quote" }
  ]
}

응답

HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": "new-uuid-generated-by-server",
  "title": "새로운 큐레이션",
  ...
}

3.5. 큐레이션 수정

기존 큐레이션의 내용을 업데이트합니다.

요청

PUT /api/v1/admin/curations/{curationId} HTTP/1.1
Host: i13d202.p.ssafy.io
Content-Type: application/json

{
  "title": "수정된 큐레이션 제목",
  "description": "수정된 큐레이션 설명입니다.",
  "items": [
    { "id": 102, "type": "book" },
    { "id": 208, "type": "quote" }
  ]
}

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "{curationId}",
  "title": "수정된 큐레이션 제목",
  ...
}

3.6. 큐레이션 삭제

특정 큐레이션을 서버에서 삭제합니다.

요청

DELETE /api/v1/admin/curations/{curationId} HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 204 No Content

4. 통계 API

방문자 분석 대시보드에 필요한 모든 데이터를 한 번에 조회합니다.

요청

GET /api/v1/admin/analytics/visitors?period=daily HTTP/1.1
Host: i13d202.p.ssafy.io

응답

HTTP/1.1 200 OK
Content-Type: application/json

{
  "realtimeVisitors": 25,
  "todayVisitors": { "value": 1250, "change": 8.2 },
  "pageViews": { "value": 18300, "change": 15.3 },

  ],
  "weeklyVisitors": [
    { "week": "1주차", "visitors": 8500 }

}
