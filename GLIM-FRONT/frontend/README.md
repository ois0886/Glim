# 📚 글:림 관리자 대시보드 프로젝트 가이드

안녕하세요! 이 프로젝트는 '글:림' 서비스의 관리자 대시보드입니다. 관리자가 서비스의 다양한 데이터를 확인하고, 사용자 및 콘텐츠를 효율적으로 관리할 수 있도록 도와주는 웹 애플리케이션이에요.

이 대시보드는 최신 웹 기술인 **Next.js**와 아름다운 UI 컴포넌트 라이브러리인 **shadcn/ui**를 사용하여 만들어졌습니다. 코딩이 처음이시더라도 걱정 마세요! 이 가이드를 통해 프로젝트를 쉽게 이해하고 실행할 수 있도록 자세히 설명해 드릴게요.

## ✨ 기술 스택 (이 프로젝트를 만든 기술들)

이 프로젝트는 다음과 같은 주요 기술들을 사용해서 개발되었습니다.

*   **Next.js**: React 기반의 웹 애플리케이션을 빠르고 효율적으로 만들 수 있게 도와주는 프레임워크입니다. 페이지 라우팅, 서버 사이드 렌더링 등 복잡한 기능을 쉽게 구현할 수 있어요.
*   **React**: 사용자 인터페이스(UI)를 구축하기 위한 JavaScript 라이브러리입니다. 웹 페이지의 각 부분을 '컴포넌트'라는 작은 조각으로 나누어 개발할 수 있게 해줍니다.
*   **TypeScript**: JavaScript에 '타입' 개념을 추가한 언어입니다. 코드를 작성할 때 발생할 수 있는 오류를 미리 잡아주고, 코드의 가독성과 유지보수성을 높여줍니다.
*   **Tailwind CSS**: 빠르고 유연하게 웹 디자인을 할 수 있도록 도와주는 CSS 프레임워크입니다. 미리 정의된 유틸리티 클래스를 조합하여 스타일을 적용합니다.
*   **shadcn/ui**: 아름답고 접근성 높은 UI 컴포넌트들을 제공하는 라이브러리입니다. 이 프로젝트의 버튼, 입력창, 표 등 대부분의 시각적인 요소들이 이 라이브러리를 기반으로 만들어졌습니다.

## 🚀 시작하기 (내 컴퓨터에서 프로젝트 실행하기)

이 섹션에서는 이 프로젝트를 여러분의 컴퓨터(로컬 환경)에서 직접 실행하는 방법을 단계별로 안내합니다.

### 🛠️ 사전 준비 (필요한 도구 설치)

이 프로젝트는 특정 버전의 Node.js에서 가장 잘 작동하도록 설계되었습니다. 여러 프로젝트를 다루거나 Node.js 버전을 유연하게 관리하고 싶다면, **Node.js 버전 관리자**를 사용하는 것을 강력히 추천합니다.

1.  **Node.js 버전 관리자 설치 (nvm)**
    *   **nvm**은 Node.js 버전을 쉽게 설치하고 전환할 수 있게 해주는 도구입니다.
    *   **Windows 사용자**:
        *   [nvm-windows 릴리즈 페이지](https://github.com/coreybutler/nvm-windows/releases)에서 최신 `nvm-setup.zip` 파일을 다운로드하여 설치를 진행하세요. 설치 마법사의 지시에 따르면 됩니다.
    *   **macOS / Linux 사용자**:
        *   터미널(명령 프롬프트)을 열고 아래 명령어를 실행하여 nvm을 설치합니다.
        ```bash
        curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
        ```
        > **설치 후 중요!**: nvm 설치가 완료되면, 터미널을 완전히 닫았다가 다시 열거나, 컴퓨터를 재시작해야 nvm 명령어를 사용할 수 있습니다.

2.  **필요 도구 확인**
    *   **Node.js**: nvm을 사용하면 `.nvmrc` 파일에 명시된 `v22.x` 버전을 자동으로 설치하고 사용하게 됩니다. Node.js는 JavaScript 코드를 컴퓨터에서 실행할 수 있게 해주는 환경입니다.
    *   **npm**: Node.js를 설치하면 `npm`(Node Package Manager)도 함께 설치됩니다. `npm`은 프로젝트에 필요한 외부 라이브러리(패키지)들을 설치하고 관리해주는 도구입니다.

### ⚙️ 설치 및 실행 방법

이제 프로젝트를 다운로드하고 실행해 봅시다!

1.  **프로젝트 다운로드 (Git 사용)**
    *   만약 이 프로젝트를 Git 저장소에서 클론(다운로드)했다면, 해당 프로젝트 폴더로 이동합니다.
    *   아직 다운로드하지 않았다면, Git이 설치되어 있다는 가정 하에 다음 명령어로 프로젝트를 클론할 수 있습니다:
        ```bash
        git clone [프로젝트_저장소_URL]
        cd [프로젝트_폴더_이름]
        ```
    *   (현재 이 문서를 보고 계시다면 이미 프로젝트 폴더 안에 있을 것입니다.)

2.  **Node.js 버전 설정 (nvm 사용 시)**
    *   프로젝트 폴더(이 `README.md` 파일이 있는 곳)로 이동한 후, 터미널에서 아래 명령어를 실행합니다.
    *   이 명령어는 `.nvmrc` 파일에 지정된 Node.js 버전(예: `v22.x`)을 자동으로 설치하고, 현재 터미널에서 해당 버전을 사용하도록 설정해 줍니다.
    ```bash
    nvm install  # .nvmrc에 지정된 Node.js 버전이 없으면 설치합니다.
    nvm use      # 현재 터미널에서 해당 Node.js 버전을 사용하도록 설정합니다.
    ```

3.  **환경 변수 설정 (`.env.local` 파일)**
    *   프로젝트가 외부 서비스(예: API 서버)와 통신하기 위해 필요한 비밀 정보(API 키, 데이터베이스 주소 등)는 `.env.local` 파일에 저장됩니다. 이 파일은 Git에 의해 관리되지 않으므로 안전하게 비밀 정보를 보관할 수 있습니다.
    *   프로젝트 루트(이 `README.md` 파일이 있는 곳)에 있는 `.env.example` 파일을 복사하여 `.env.local` 파일을 생성합니다.
    ```bash
    cp .env.example .env.local
    ```
    *   `cp` 명령어는 파일을 복사하는 명령어입니다. Windows에서는 `copy .env.example .env.local`을 사용할 수도 있습니다.
    *   `.env.local` 파일을 열어 필요한 환경 변수(예: `NEXT_PUBLIC_API_URL=http://localhost:8080`)를 설정해 주세요. 어떤 변수를 설정해야 하는지는 `.env.example` 파일에 예시로 나와 있습니다.

4.  **의존성 라이브러리 설치**
    *   프로젝트가 작동하는 데 필요한 모든 외부 라이브러리(패키지)들을 설치합니다. 이 라이브러리 목록은 `package.json` 파일에 정의되어 있습니다.
    ```bash
    npm install
    ```
    *   이 명령어를 실행하면 `node_modules`라는 폴더가 생성되고, 그 안에 필요한 모든 라이브러리 파일들이 다운로드됩니다.

5.  **개발 서버 실행**
    *   이제 프로젝트를 웹 브라우저에서 볼 수 있도록 개발 서버를 실행합니다.
    ```bash
    npm run dev
    ```
    *   이 명령어를 실행하면 Next.js 개발 서버가 시작되고, 코드 변경 사항이 있을 때마다 자동으로 웹 페이지에 반영됩니다.

6.  **브라우저에서 확인**
    *   터미널에 "ready - started server on http://localhost:3000"과 같은 메시지가 나타나면, 웹 브라우저를 열고 다음 주소로 접속하세요:
    *   [http://localhost:3000](http://localhost:3000)
    *   이제 '글:림' 관리자 대시보드의 첫 화면을 볼 수 있을 거예요!

│   │   └── ... (다양한 UI 컴포넌트 파일들)
│   └── theme-provider.tsx # 웹사이트의 테마(다크 모드/라이트 모드)를 관리하는 컴포넌트

## 📂 주요 폴더와 파일 상세 설명

### 📁 `app/` 폴더

*   **`globals.css`**: 이 파일에 작성된 CSS 스타일은 웹사이트의 모든 페이지에 공통으로 적용됩니다. 전역적인 디자인 규칙을 정의할 때 사용해요.
*   **`layout.tsx`**: 웹사이트의 전체적인 '뼈대'를 정의하는 파일입니다. 모든 페이지가 공유하는 헤더, 푸터, 내비게이션 바, 그리고 웹 폰트 설정 등이 이곳에 포함됩니다.
*   **`loading.tsx`**: 사용자가 페이지를 이동하거나 데이터를 불러오는 동안, 화면에 '로딩 중...'과 같은 메시지나 애니메이션을 보여주고 싶을 때 이 파일을 사용합니다.
*   **`page.tsx`**: 웹사이트의 메인 대시보드(첫 화면)를 구성하는 파일입니다. 이곳에서 다양한 관리자 섹션 컴포넌트들을 불러와 배치합니다.

### 📁 `components/` 폴더

이 폴더는 웹사이트를 구성하는 작은 '부품'들, 즉 재사용 가능한 UI 컴포넌트들을 모아둔 곳입니다.

*   **`admin/`**: 관리자 대시보드의 핵심 기능들을 담당하는 컴포넌트들이 모여 있습니다. 각 컴포넌트는 사이드바 메뉴의 특정 항목과 연결되어 있습니다.
    *   `app-sidebar.tsx`: **웹사이트 왼쪽의 메뉴(사이드바) 컴포넌트**. 관리자 대시보드의 각 섹션으로 이동할 수 있는 링크들이 여기에 포함됩니다.
    *   `curation-editor.tsx`: **큐레이션 생성 및 편집기** (`사이드바: 큐레이션 편집`). 관리자가 '인용구'와 '도서'를 조합하여 하나의 주제를 가진 콘텐츠 묶음, 즉 '큐레이션'을 만들거나 기존 큐레이션을 수정할 수 있는 페이지입니다. 예를 들어, "사랑에 대한 명언과 관련 도서" 큐레이션을 여기서 만들 수 있어요.
    *   `curation-list.tsx`: **큐레이션 목록 페이지** (`사이드바: 큐레이션 편집`과 연관). `curation-editor.tsx`에서 생성된 모든 큐레이션의 목록을 보여주고, 각 큐레이션을 클릭하여 상세 내용을 보거나 편집할 수 있도록 연결해 줍니다.
    *   `post-management.tsx`: **게시물 관리 페이지** (`사이드바: 게시물 관리`). 사용자들이 서비스에 작성한 게시물(예: 독서 감상문, 서평)들을 관리하는 페이지입니다. 부적절한 게시물을 삭제하거나, 특정 게시물을 검색하는 등의 기능을 수행합니다.
    *   `sortable-list-wrapper.tsx`: **정렬 가능한 리스트 래퍼 컴포넌트**. 드래그 앤 드롭 등으로 목록의 순서를 변경할 수 있는 기능을 제공하는 컴포넌트입니다. 주로 큐레이션이나 게시물 목록의 순서를 관리할 때 사용될 수 있습니다. (사이드바에 직접 연결되지 않음)
    *   `user-demographics.tsx`: **사용자 통계 시각화 페이지** (`사이드바: 사용자 통계`). 서비스에 가입한 사용자들의 인구통계학적 데이터(예: 연령대, 성별 분포)를 그래프나 차트 형태로 시각적으로 보여주는 컴포넌트입니다. 서비스의 주요 사용자층을 파악하는 데 도움이 됩니다.
    *   `user-graphics.tsx`: **사용자 그래픽 데이터 시각화 컴포넌트**. 사용자 관련 데이터를 다양한 그래픽(차트, 그래프 등)으로 시각화하여 보여주는 컴포넌트입니다. 예를 들어, 사용자 활동량 추이, 인기 있는 콘텐츠 유형 등을 시각적으로 분석할 수 있습니다. (사이드바에 직접 연결되지 않음)
    *   `user-management.tsx`: **사용자 계정 관리 페이지** (`사이드바: 사용자 관리`). 서비스에 가입한 모든 사용자의 목록을 보여주고, 각 사용자의 정보를 확인하거나, 계정을 정지시키거나, 권한을 변경하는 등의 작업을 할 수 있는 페이지입니다.
    *   `visitor-analytics.tsx`: **방문자 분석 페이지** (`사이드바: 방문자 분석`). 웹사이트의 트래픽(방문자 수, 페이지 뷰 등)과 방문자 행동(어떤 페이지를 많이 보는지 등)을 분석하여 보여주는 대시보드입니다. 서비스의 인기도와 사용자들의 관심사를 파악하는 데 유용합니다.
*   **`ui/`**: `shadcn/ui` 라이브러리를 통해 생성된 기본적인 UI 컴포넌트들(버튼, 입력창, 체크박스, 다이얼로그 등)이 모여 있습니다. 이 컴포넌트들은 프로젝트 전반에서 재사용됩니다.
*   **`theme-provider.tsx`**: 웹사이트의 전체적인 '테마'를 관리하는 컴포넌트입니다. 예를 들어, 사용자가 '다크 모드'나 '라이트 모드'를 선택했을 때 웹사이트의 색상 팔레트가 자동으로 변경되도록 설정하는 기능을 제공합니다.

### 📁 `hooks/` 폴더

React의 특별한 기능인 'Hook'들을 모아둔 곳입니다. Hook은 함수형 컴포넌트에서 상태(state)나 생명주기(lifecycle) 기능을 사용할 수 있게 해줍니다.

*   **`use-mobile.tsx`**: 현재 웹사이트에 접속한 기기가 모바일 환경인지(스마트폰, 태블릿 등)를 감지하는 기능을 제공하는 Hook입니다. 이를 통해 모바일 환경에 맞는 반응형 디자인을 적용할 수 있습니다.
*   **`use-toast.ts`**: 웹사이트 하단이나 상단에 잠시 나타났다가 사라지는 작은 알림 메시지(토스트 메시지)를 띄우는 기능을 제공하는 Hook입니다. 사용자에게 작업 완료나 오류 발생 등을 알릴 때 유용합니다.

### 📁 `lib/` 폴더

프로젝트 전반에서 공통으로 사용되는 유틸리티 함수(도우미 함수)들을 모아둔 곳입니다.

*   **`utils.ts`**: 다양한 유틸리티 함수들이 정의되어 있습니다. 예를 들어, 날짜 형식을 변경하거나, 문자열을 특정 방식으로 처리하는 등 여러 곳에서 반복적으로 사용되는 코드들을 여기에 모아두어 코드의 재사용성을 높입니다.

### 📁 `public/` 폴더

이미지, 아이콘, 샘플 데이터 등 웹 브라우저에서 직접 접근할 수 있는 정적 파일들을 보관하는 곳입니다.

*   **`placeholder-*.png/svg/jpg`**: 아직 실제 이미지가 준비되지 않았을 때 임시로 사용하는 '자리 표시자(placeholder)' 이미지 파일들입니다.
*   **`users.json`**: 개발 단계에서 테스트용으로 사용되는 샘플 사용자 데이터가 JSON 형식으로 저장되어 있습니다.

### 🗄️ 주요 파일들

*   **`.gitignore`**: Git(버전 관리 시스템)이 프로젝트를 관리할 때, 특정 파일이나 폴더(예: `node_modules`, `.env.local` 등)는 추적하지 않도록 설정하는 파일입니다. 비밀 정보나 빌드 결과물 등이 Git 저장소에 올라가지 않도록 방지합니다.
*   **`API 명세서.html`**: 이 프로젝트가 외부 API 서버와 어떻게 통신하는지에 대한 규칙과 정보를 담고 있는 문서입니다. 어떤 데이터를 주고받는지, 어떤 주소로 요청해야 하는지 등이 설명되어 있습니다.
*   **`components.json`**: `shadcn/ui` 컴포넌트와 관련된 설정 정보가 담겨 있을 수 있습니다.
*   **`GEMINI.md`**: 이 프로젝트와 관련된 Gemini(구글 AI)의 설정이나 참고 사항이 기록된 문서입니다.
*   **`next-env.d.ts`, `next.config.mjs`**: Next.js 애플리케이션의 동작 환경과 설정을 정의하는 파일들입니다. Next.js가 프로젝트를 어떻게 빌드하고 실행할지 등을 설정합니다.
*   **`package.json`**: 프로젝트의 '설명서'와 같은 파일입니다. 프로젝트의 이름, 버전, 개발에 필요한 모든 라이브러리 목록(`dependencies`, `devDependencies`), 그리고 `npm run dev`와 같은 실행 명령어(`scripts`)들이 정의되어 있습니다.
*   **`package-lock.json`, `pnpm-lock.yaml`**: `npm install` 명령어를 실행했을 때, 설치된 라이브러리들의 정확한 버전 정보와 의존성 관계를 기록하는 파일입니다. 이 파일들 덕분에 다른 개발자가 프로젝트를 설치해도 동일한 버전의 라이브러리들이 설치되어 환경에 따른 오류를 줄일 수 있습니다. (자동 생성되므로 직접 수정할 필요는 없습니다.)
*   **`postcss.config.mjs`**: CSS(스타일)를 브라우저가 이해하기 쉽게 변환하거나, 특정 기능을 추가하는 '전처리' 도구인 PostCSS의 설정 파일입니다.
*   **`tailwind.config.ts`**: Tailwind CSS 프레임워크의 설정 파일입니다. Tailwind CSS의 색상 팔레트, 폰트 크기, 간격 등 디자인 시스템을 커스터마이징할 때 사용합니다.
*   **`tsconfig.json`**: TypeScript 컴파일러의 설정 파일입니다. TypeScript 코드를 JavaScript로 변환할 때 어떤 규칙을 적용할지 등을 정의합니다.

## 📜 사용 가능한 스크립트 (자주 쓰는 명령어)

`package.json` 파일의 `scripts` 섹션에 정의된 명령어들입니다. 터미널에서 `npm run [스크립트_이름]` 형식으로 실행할 수 있습니다.

*   `npm run dev`: **개발 모드 시작**. 이 명령어를 실행하면 Next.js 개발 서버가 시작되고, 웹 브라우저에서 프로젝트를 확인할 수 있습니다. 코드를 수정하면 웹 페이지에 실시간으로 변경 사항이 반영되어 개발할 때 가장 많이 사용합니다.
*   `npm run build`: **배포용 빌드**. 이 명령어를 실행하면 프로젝트의 코드를 웹 서버에 배포할 수 있는 최적화된 형태로 변환(빌드)합니다. 실제 서비스에 올리기 전에 이 과정을 거쳐야 합니다.
*   `npm run start`: **빌드된 애플리케이션 실행**. `npm run build`로 빌드된 결과물을 실행합니다. 개발 서버(`npm run dev`)와 달리, 이 명령어는 실제 서비스 환경처럼 최적화된 상태로 애플리케이션을 실행합니다.
*   `npm run lint`: **코드 스타일 검사**. 이 명령어를 실행하면 ESLint라는 도구를 사용하여 프로젝트 코드의 스타일을 검사하고, 잠재적인 오류나 비효율적인 코드를 찾아줍니다. 코드의 품질을 유지하고 팀원 간의 코드 스타일을 통일하는 데 중요합니다.
