## 🛠️ 기술적 도전
### Frontend

#### 1. 성능 최적화와 사용자 경험 개선
- **이미지 렌더링 성능 문제 해결**  
  - 다양한 이미지 vs 소수 이미지를 반복해서 뿌릴 때 브라우저 성능 차이를 실험적으로 비교  
  - 같은 이미지를 반복적으로 사용하는 편이 렌더링 부하가 적다는 점을 확인하고 최적화 방향으로 반영  

- **3D 갤러리 UX 개선**  
  - 초기에는 스크롤 시 버벅임과 화면 밀도가 낮은 문제가 있었음  
  - 관성 스크롤 로직과 자동 간격 조절, 카메라 거리 계산으로 해결 → 더 매끄러운 UX 제공  

#### 2. 발표 자료와 시연 환경의 한계 극복
- 발표 자료가 스크립트에 비해 시각적으로 단조롭고 불필요한 텍스트가 많다는 피드백을 받음  
- **개선 방안**  
  - 키워드 중심의 자료 구성으로 전환, 장표 수는 늘리되 빠른 템포로 진행  
  - 영상 대신 PPT 장표로 세분화하여 시연 상황을 시각적으로 보완  
  - 불필요한 애니메이션/색상은 줄이고, 핵심 데이터(예: “6만 5천 권”)는 강조 효과를 통해 짧게 전달  

#### 3. 비전공자로서의 성장 도전
- 짧은 기간(반 학기) 동안 학습과 취업 준비를 병행해야 하는 상황  
- **집중했던 역량**  
  - React & TypeScript 기반의 프론트엔드 기본기, 문제 해결 능력  
  - 코딩 테스트 대비를 통한 논리적 사고력 향상  
  - 프로젝트를 기록으로 남겨 포트폴리오에 활용  
- 단순히 개발에 국한되지 않고, 기획서 작성·화면 설계·UX 분석까지 경험하며 기획과 개발을 아우르는 역량을 준비  
- “한 분야(프론트엔드)를 깊게 파되, 기획/UX까지 이해하는 균형 잡힌 준비”를 스스로의 목표로 삼음
